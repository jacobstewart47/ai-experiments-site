A color contrast analyser tool

By Maria Lopez

Cursor Claude Github

Introducing ColorSight, an innovative accessibility Chrome plugin designed specifically for designers. ColorSight enhances your web design experience by providing real-time color contrast analysis and suggestions to ensure compliance with WCAG standards.

How did you build it?

I built it as a Chrome extension using plain HTML, CSS, and JavaScript, just to keep it lightweight. I used the Chrome APIs to scan the DOM for common accessibility issues—things like missing alt text, low contrast, or improper heading structure. The UI was really simple: a sidebar that would highlight issues on the page and explain what was wrong and how to fix it. I didn’t use any frameworks because I wanted to really understand how everything worked under the hood.

How long did it take?

Roughly a couple of weekends, maybe 15–20 hours all in. It started off as a quick side project but I kept tweaking bits here and there as I learned more about accessibility guidelines and the Chrome extension docs. It probably could’ve been faster, but I was figuring stuff out as I went.

What was the biggest challenge?

Getting reliable contrast ratio calculations was weirdly fiddly. The maths for checking WCAG contrast levels is a bit obscure, and getting accurate colour values from computed styles in the browser took some trial and error. Also, just wrapping my head around the quirks of the DOM when you’re injecting scripts into third-party pages — some elements behave in unexpected ways.

What was the biggest learning?

Honestly, just how many accessibility issues are invisible if you’re not actively looking for them. Even sites that seem “clean” visually can be a nightmare for screen readers. It really made me appreciate how much tooling can help designers and devs catch stuff early. Also, building the tool myself made the guidelines stick in my head way more than just reading about them.
